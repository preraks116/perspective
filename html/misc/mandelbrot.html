Mandelbrot Set

Simulation

Here is a simulation of the Mandelbrot set slowly taking shape. With each frame, the number of iterations over which each point is given a color is increase from 1 to 30, and as the number of iterations increases, the detail of the set increases.

The big shape that you see in the centre of the set is called the main cardioid. The cardioid is the shape that is the closest to the origin. To its left is the shape which is called the main disk.

Explanation

This is a simulation of the Mandelbrot set. The set is a fractal that is a collection of points in two-dimensional space that are generated by a sequence of iterations of a function. Fractals are special because of their recursive pattern where upon zooming into a a section of the graph, the mini version of the fractal is generated.

The Mandelbrot set, known as the land of finite land and infinite coast, is a set of points in the complex plane that are generated by a sequence of iterations of a function that is called the Mandelbrot set. The function is defined as:

zn+1 = zn2 + cn

where zn is the value of zn at the nth iteration, cn is a complex number, and n is the number of iterations.

The output from this function is taken and put back into the function and this is done for a certain number of iterations. If the value of the output after the iterations continues to increase in value , then it is not a part of the Mandelbrot Set.

If the output doesnt grow out of bounds as we continue to iterate, then it is a part of the Mandelbrot Set.

Time Complexity

Let the total number of pixels be n and the number of maximum iterations be m.

Then the time complexity of the algorithm is O(n*m). This is because for each pixel, we have to iterate m times and then evaluate the output to check if its part of the set or not.

Code


for (var i = 0; i < width; i++) { 
    // here width is the width of the canvas
    for (var j = 0; j < height; j++) {
        // here height is the height of the canvas
        //calculate the real and imaginary components
        var a = map(i - 70, 0, width, -2.5, 2.5);
        var b = map(j, 0, height, -2.5, 2.5);
        //calculate the initial real and imaginary components
        var ca = a;
        var cb = b;
        //calculate the number of iterations
        var n = 0;
        //iterate until the number of iterations reaches the max
        while (n < maxiterations) {
            //calculate the real and imaginary components
            var aa = a * a - b * b;
            var bb = 2 * a * b;
            #calculate the real and imaginary components
            a = aa + ca;
            b = bb + cb;
            //check if the number of iterations has reached the max
            if (a * a + b * b > 16) {
                break;
            }
            n++;
        }
        //set the color of the pixel
        var bright = map(n, 0, maxiterations, 0, 255);
        bright = constrain(bright, 0, 255);
        var pix = (i + j * width) * 4;
        pixels[pix + 0] = bright;
        pixels[pix + 1] = bright;
        pixels[pix + 2] = bright;
        pixels[pix + 3] = 255;
    }
}

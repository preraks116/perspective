Conway's Game of Life

Simulation

As you can see, above is a simulation of the game of life. The simulation is a cellular automaton where the state of the board (array of cells) depends on the previous state of the board.

You can some cells that were initially alive might not be alive anymore, but some groups of cells have seemed to have formed a stable pattern.

Some of the patterns that you can see are:

  • Block : A square the consists of cells that are alive.
  • Beehive : A pattern with 6 cells that are alive which looks slightly like a hexagon.
  • Boat : A pattern with 5 cells that are alive which looks somewhat like a small fish.
  • Blinker : This is an oscillating pattern with period 2 which is essentially a line of 3 cells that are alive osciillating between vertical and horizontal. It looks like a plus because of how fast the oscillations are.
  • Tub : This is a pattern with 4 cells that are alive which looks like a plus sign with a hole in the middle.

Explanation

The Game of Life is a cellular automaton devised by the British mathematician John Horton Conway in 1970.

It consists of a board of cells. Each cell is either alive or dead which represented by its color, white or black, respectively.

At each stage, the board is updated based the previous state of the board.

The rules of the game are as follows:

  • Any live cell with fewer than two live neighbours dies, as if caused by under-population.
  • Any live cell with two or three live neighbours lives on to the next generation.
  • Any live cell with more than three live neighbours dies, as if by overcrowding.
  • Any dead cell with exactly three live neighbours becomes a live cell, as if by reproduction.

The initial pattern of the board is known as the seed of the board and each subsequent board permuatation is called a generation.

Time Complexity

Let the number of cells be n. Then the time complexity of the algorithm is O(n) as the algorithm looks at each cells neighbours one by one do determine whether the cell is now alive or dead.

function make2DArray(cols, rows) {
    let arr = new Array(cols);
    for (let i = 0; i < arr.length; i++) {
        arr[i] = new Array(rows);
    }
    return arr;
}

let grid;
let cols;
let rows;
let resolution = 10;

function setup() {
    createCanvas(720, 400);
    cols = width / resolution;
    rows = height / resolution;

    grid = make2DArray(cols, rows);
    for (let i = 0; i < cols; i++) {
        for (let j = 0; j < rows; j++) {
            grid[i][j] = floor(random(2));
        }
    }
}

function draw() {
    background(0);

    for (let i = 0; i < cols; i++) {
        for (let j = 0; j < rows; j++) {
            let x = i * resolution;
            let y = j * resolution;
            if (grid[i][j] == 1) {
                fill(255);
                stroke(0);
                rect(x, y, resolution - 1, resolution - 1);
            }
        }
    }

    let next = make2DArray(cols, rows);

    // Compute next based on grid
    for (let i = 0; i < cols; i++) {
        for (let j = 0; j < rows; j++) {
            let state = grid[i][j];
            // Count live neighbors!
            let sum = 0;
            let neighbors = countNeighbors(grid, i, j);

            if (state == 0 && neighbors == 3) {
                next[i][j] = 1;
            } else if (state == 1 && (neighbors < 2 || neighbors > 3)) {
                next[i][j] = 0;
            } else {
                next[i][j] = state;
            }
        }
    }

    grid = next;
}

function countNeighbors(grid, x, y) {
    let sum = 0;
    for (let i = -1; i < 2; i++) {
        for (let j = -1; j < 2; j++) {
            let col = (x + i + cols) % cols;
            let row = (y + j + rows) % rows;
            sum += grid[col][row];
        }
    }
    sum -= grid[x][y];
    return sum;
}

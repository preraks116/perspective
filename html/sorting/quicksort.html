Divide and Conquer

Quick Sort

Simulation

Quick Sort is a divide and conquer algorithm. It picks an element as pivot and partitions the given array around the picked pivot. There are many different versions of quick sort. Most of the times, it picks the first element as pivot, but there are many other possibilities.

In the simulation that you see above, the green bars represent the unsorted list of numbers. The height of the bars corresponds to the value of the number in the list. The red bars represent the pivots that are used in the algorithm. The height of the red bars corresponds to the value of the pivot.

Explanation

Quick Sort is an algorithm which is most commonly known as a divide and conquer algorithm.

The algorithm works as follows: It starts of by first selecting one of the entries in the array as a 'pivot' element, which is used as reference for comparison's sake. A pointer is initiated at the first element of the array, let us call it the 'left' pointer. Similarly a pointer is initiated at the last element of the array and we call this as the 'right' pointer.

We start moving the left pointer to the right by one element and the right pointer to the left by one element. When the value which the left pointer indicates is less than the pivot value, this process of moving the left pointer keeps happening till value is greater than or equal to pivot. Similarly, when the value which the right pointer indicates is greater than the pivot value, it's process also keeps happening till the value is less than or equal to the pivot.

When the left pointer is less than or equal to the right pointer, the location of the elements at these two pointers is swapped. The process is restarted from the beginning if the pointers don't meet.

The algorithm can be summarized as follows:

  • Choose a pivot element
  • Reorder the elements in the array so that all elements less than the pivot come before it, and all elements greater than the pivot come after it
  • Recursively apply the above steps to the sub-arrays of elements less than and greater than the pivot

Complexity

Worst Case Time Complexity of Quick Sort is O(n2).

Best Case Time Complexity of Quick Sort is O(n log n) when the array is already sorted.

Average Case Time Complexity of Quick Sort is O(n log n) when the array is randomly distributed.

Space Complexity of Quick Sort is O(log n)

# Python program for implementation of Quicksort Sort

# This function takes last element as pivot, places
# the pivot element at its correct position in sorted
# array, and places all smaller (smaller than pivot)
# to left of pivot and all greater elements to right
# of pivot
    
    
def partition(arr, low, high):
    i = (low-1)         # index of smaller element
    pivot = arr[high]     # pivot
    
    for j in range(low, high):
    
        # If current element is smaller than or
        # equal to pivot
        if arr[j] <= pivot:
    
            # increment index of smaller element
            i = i+1
            arr[i], arr[j] = arr[j], arr[i]
    
    arr[i+1], arr[high] = arr[high], arr[i+1]
    return (i+1)
    
# The main function that implements QuickSort
# arr[] --> Array to be sorted,
# low  --> Starting index,
# high  --> Ending index
    
# Function to do Quick sort
    
    
def quickSort(arr, low, high):
    if len(arr) == 1:
        return arr
    if low < high:
    
        # pi is partitioning index, arr[p] is now
        # at right place
        pi = partition(arr, low, high)
    
        # Separately sort elements before
        # partition and after partition
        quickSort(arr, low, pi-1)
        quickSort(arr, pi+1, high)

n = len(arr)

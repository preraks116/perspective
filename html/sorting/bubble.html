Bubble Sort

Simulation

The blue bars that you see represent the different numbers in a list. The height of the bars represent the size of the number, which means that the smaller the number, the shorter the bar will be and vice versa. The bars are being sorted by the bubble sort algorithm. The algorithm works by comparing two adjacent bars and swapping them if they are in the wrong order. Here the 2 white bars represent the two numbers that are being compared.

Explanation

Bubble Sort, sometimes referred to as Sinking Sort, is a simple sorting algorithm that repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. The pass through the list is repeated until the list is sorted. The algorithm, which is a comparison sort, is named for the way smaller or larger elements "bubble" or move slowly to the top of the list.

The bubble sort algorithm works the way we sort playing cards in our hands. You imagine a row of n cards where the ith card is showing the number i. You take the top card and place it in the correct position by swapping it with the card immediately above it. You then take the second card and place it in the correct position by swapping it with the card immediately above it. You continue this process until you have sorted the entire row.

The algorithm consists of two phases: the sorting phase and the checking phase. The sorting phase moves through the list looking for adjacent pairs of items that are out of order, and swaps them. The checking phase compares each pair of items to make sure they are in the correct order.

The algorithm is not efficient for large lists. It is generally only efficient for lists where the number of items is much smaller than the number of passes through the list.

The algorithm is not stable, two items may be swapped when they are compared.

The algorithm is not adaptive, it does not consider whether the list is already sorted.

The algorithm is not in-place, it requires extra space for the temporary storage of the swapped items.

Complexity

The time complexity of bubble sort is O(n2) because it has two nested loops.

The space complexity of bubble sort is O(1) because it does not require any extra space.

Code


    # Creating a bubble sort function  
    def bubble_sort(num_list):  
        # Outer loop for traverse the entire list  
        for i in range(0,len(num_list)-1):  
            for j in range(len(num_list)-1):  
                if(num_list[j]>num_list[j+1]):  
                    temp = num_list[j]  
                    num_list[j] = num_list[j+1]  
                    num_list[j+1] = temp  
        return num_list  

    num_list = [5,2,4,6,1,3]  
    print("Sorted list: ", bubble_sort(num_list))
